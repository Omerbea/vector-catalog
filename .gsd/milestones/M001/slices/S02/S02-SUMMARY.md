# S02: Product CRUD API

**Spring Boot app running in Docker Compose: POST /api/products triggers DB embedding, GET returns product or 404. Full stack verified end-to-end.**

## What Was Built

Built the complete Spring Boot application layer. Two fixes were needed during integration: `price` changed to `BigDecimal` (Hibernate rejects `scale` on floating-point types), and `SERIAL` changed to `BIGSERIAL` in `init.sql` (Hibernate `Long id` maps to `bigint`).

The `Product` entity maps the `embedding` column with `@JdbcTypeCode(SqlTypes.VECTOR)` but the application never writes to it — Spring issues a plain `save()` with no embedding, the DB trigger fires, and the vector appears in the row after the transaction commits.

Full stack verified: `docker compose up --build` starts both services in ~14s, `POST /api/products` returns 201 with correct JSON and the DB trigger generates a 768-dim Gemini embedding.
